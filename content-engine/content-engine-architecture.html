Reflection / RL Loop

The two-tier feedback system: an immediate Retry signal fires before every draft, and a deep Composite reflection fires on every rejection. Based on findings from both the Self-Reflection paper and Reflect-Retry-Reward.

Self-Reflection paper finding: Composite reflection (Explanation + Solution + Instructions) achieved 93% vs Retry's 83% — a 10-point gap. Simply retrying is the weakest form. The biggest jump comes from Solution-type reflections that work through the correction step-by-step.
Reflect-Retry-Reward finding: Train the reflection quality, not the content quality. The bottleneck is how well the model diagnoses failure — not how much content data it has seen. Scoring reflection quality separately (specific + generalizable) is the mechanism for this.

Reflection Quality Scoring Rubric

Dimension Score 1–4 (bad) Score 5–7 (ok) Score 8–10 (good)
Specificity "The hook wasn't good" "The hook was too generic" "The hook led with a concept instead of a personal moment — failed the 'me not you' rule"
Generalizability Only applies to this exact post Applies to this content type One-sentence rule that applies across all LinkedIn posts
Actionability Vague aspiration Describes what to avoid Tells the model exactly what to do differently next time

Failure Categories to Track

Category 01
Hook Failure
Hook led with concept not story · Too vague · Too AI-sounding · Missing tension · No personal stake
Category 02
Tone Failure
Sounds like a LinkedIn thought leader cliché · Over-polished · Missing Ayush's directness · Wrong register for platform
Category 03
Depth Failure
Surface-level insight · No original perspective · Restates obvious · Missing the "why this matters to me specifically"
Category 04
Format Failure
Wrong length for platform · Poor structure · CTA missing or weak · Carousel flow breaks · Thread pacing off
Category 05
Story Fit Failure
Wrong life story matched · Story doesn't support the angle · Forced connection between idea and narrative

MetaGPT finding: Adding code execution (a binary pass/fail ground truth) gave a 5.4% benchmark improvement because it's immune to hallucination — the code either runs or it doesn't. Layer 4 (AI Detection API) is your equivalent. The model cannot hallucinate its way past an external API score.

Layer-by-Layer Breakdown

Layer 1
Forbidden Phrase Check
Type: Deterministic string match

Source: voice_memory.voice_rules.forbidden_phrases

On fail: Immediate rejection. No reflection triggered — this is a rule violation, not a judgment call. The Content Producer should never have produced this; the pre-write signal updates.

Cost: Near zero. Fast.
Layer 2
Format Rules Check
Type: Deterministic rules per format

Checks: Word count in range · CTA present · Structure valid (thread has numbered points, carousel has slide count, etc.)

On fail: Rejection with category format_failure. Routes to reflector. Reflector generates instructions for format compliance.

Cost: Near zero. Logic-based.
Layer 3
Tone Match Score
Type: LLM-judged (one API call)

Prompt: Score this draft 0–1 against these voice rules and hook rules. Return JSON: {score, reason, specific_issue}

Threshold: ≥ 0.75 to pass

On fail: Routes to reflector with the specific issue logged. Tone scores tracked in rl_metrics over time.

Cost: ~$0.001 per check.
Layer 4
AI Detection API
Type: External API call — executable ground truth

Threshold: AI probability ≤ 0.25 to pass

On fail: Score logged to ai_detection_log in voice-memory. Routes to reflector. Reflector generates instructions specifically about writing patterns that triggered detection.

Why last: Most expensive check. Only runs if draft passes layers 1–3. Catches the drafts that sound like Ayush but still pattern-match as AI.
