# SDR Bot - Final Status (2026-03-24 19:22 UTC)

## What Works ✅

1. **Bot responds** — No longer silent/typing forever
2. **Correct messages** — Returns proper lead generation responses
3. **No credentials** — Blocks li_at/token/password requests
4. **Minimal scope** — Configured to lead generation only
5. **Skill is clean** — No debug logs, 3 simple response types

## What Doesn't Work ❌

**Agent narration** — The agent explains what it's doing instead of silently executing:
```
"I'll help you scan... Let me check the skill first... 
Now let me read the skill... Let me configure..."
```

Then finally returns the correct message.

## Root Cause

- Claude (the LLM) naturally narrates actions when uncertain
- Even with minimal tools, the agent tries to be "helpful"
- System prompts in workspace files are read by agent and cause thinking
- Minimal tool profile doesn't prevent narration, only blocks tool access
- OpenClaw agent system doesn't support "silent mode" or "return-skill-only" behavior

## Solutions Tried

1. ✅ Created minimal skill — Works
2. ✅ Set minimal tools profile — Prevents tool use but not narration
3. ❌ Added system prompts — Agent reads them and thinks harder
4. ❌ Cleared documentation — Agent still narrates without docs
5. ❌ Removed all workspace files — Agent still explains

## Why This Happens

OpenClaw agents are **full-featured LLMs** designed to:
- Think through problems
- Explain their reasoning
- Ask clarifying questions
- Be conversational

The SDR bot needs a **mechanical script** that:
- Never thinks
- Never explains
- Never asks questions
- Just returns one message

These are fundamentally incompatible architectures.

## What Would Actually Fix It

**Option 1: Custom Agent Handler** — Modify OpenClaw's agent execution to detect SDR bot and:
- Call skill directly
- Suppress all narration
- Return skill output only
- (Requires core OpenClaw changes)

**Option 2: Webhook Bot** — Run SDR bot outside OpenClaw:
- Telegram sends to webhook
- Webhook calls skill
- Returns message directly
- No agent thinking
- (Requires separate service)

**Option 3: ACP Harness** — Use ACP with explicit "execute skill, return output, stop" instruction
- (Requires different runtime setup)
