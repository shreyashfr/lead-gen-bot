# 2026-04-01 — Session Notes

## LinkedIn Lead Gen Bot — Major Refactor

### Sales Nav → Voyager API Migration
- Removed Sales Navigator dependency entirely
- Renamed skill from `linkedin-salesnav` to `linkedin-leads`
- Now uses LinkedIn Voyager API with just `li_at` cookie (no `li_a` or enterprise tokens needed)
- JSESSIONID is auto-fetched from LinkedIn on each request (used as csrf-token header)

### Key Discovery: Company-First Search
- **Problem**: Voyager keyword search returns people who mention keywords *anywhere* in profile, not necessarily current employees
- **Solution**: Company-first approach:
  1. Search for company → get company ID (e.g., Stripe = 2135371)
  2. Use `currentCompany` filter in people search
  3. Returns **verified employees** with **actual current titles**

### New Scripts
- `skills/linkedin-leads/scripts/pipeline.js` — keyword search (Voyager primary, Serper fallback)
- `skills/linkedin-leads/scripts/company-leads.js` — **company-first search** (recommended)
- `lead-gen-bot/scripts/salesnav-scraper.js` — updated to use Voyager API
